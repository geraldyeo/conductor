Section 01
Graph Structure
An adjacency list where each node is a session status and each edge is a guarded transition. Edges carry a precedence number — lower wins. Wildcard (global) edges are appended to every non-terminal node at build time.

Section 03
Graph Construction & Validation
The graph is built from a declarative, flat list of edge declarations — one line per transition in the PRD. Compiled into the adjacency list at startup.

★ Key Insight
The validation layer turns the PRD's transition table from a spec you hope the code matches into an invariant the system enforces. Invalid graphs are rejected at startup.
Section 04
Poll Loop Architecture
The orchestrator's heartbeat. Runs on a fixed 30s interval with re-entrancy protection. Three phases per tick — cleanly separated read/compute/write.
Gather
Read · I/O heavy
Collect world state into PollContext. Parallelized across sessions (max 5). Sequential within each session.
→
Evaluate
Compute · Pure
Walk graph edges in precedence order. First matching guard wins. Zero I/O. O(out-degree).
→
Transition
Write · Side effects
Atomic status update, journal append, fire entry actions. Sequential to avoid races.
Re-entrancy guard:  If a tick is still in-flight, the next scheduled tick is a no-op. No overlapping cycles.
★ Key Insight
The gather→evaluate→transition pipeline maps to read/compute/write. The pure evaluate phase can be property-tested exhaustively — generate random PollContexts and verify the graph always produces valid transitions.
Section 05
Side Effects & Crash Recovery
Entry actions are keyed by target status — not by edge. A session entering killed always destroys its runtime, regardless of which transition got it there.

Crash Recovery — No Special Codepath
1. Load all non-terminal sessions from metadata files 2. Run one immediate poll tick (gather → evaluate → transition) 3. Graph + live world state naturally converge to correct status
No event log replay. The poll loop is the recovery mechanism.
★ Key Insight
Crash recovery is "just run the poll loop" — a direct consequence of the state machine being deterministic and gathering fresh world state each tick. No event log to replay, no separate recovery codepath. The graph evaluates current reality and converges.
